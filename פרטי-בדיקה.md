# פרטי בדיקה – פרויקט Babysitter

מסמך זה מרכז את המידע שהבודקים צריכים לפני בדיקת הפרויקט.

## הרצה מקומית

1. בתיקיית `server` להריץ:
   ```
   npm install
   npm start
   ```
2. השרת עולה על `http://localhost:3000` ומגיש גם את קבצי הלקוח.
3. דף הבית: `http://localhost:3000/index.html`

> חשוב: יש לגשת לאתר דרך כתובת השרת (`http://localhost:3000/...`) ולא לפתוח את קובצי ה-HTML ישירות (file://), אחרת הקריאות ל-API לא יעבדו.

## API חיצוני

- **Nominatim (OpenStreetMap) – Geocoding**: המרת כתובת בעברית ל-קואורדינטות (lat/lng).
- נקודת הקצה בשרת: `GET /api/geo/geocode?address=...` (ראו `server/controllers/geoController.js`).
- שימוש אמיתי במערכת: מיקום בייביסיטרים על המפה.
- בנוסף, הלקוח משתמש ב-OpenStreetMap Tiles להצגת המפה עצמה.

## ספריית JavaScript

- **Leaflet 1.9.4** – הצגת מפות אינטראקטיביות, פינים מותאמים ובחירת בייביסיטר על המפה (דפי החיפוש והמשמרת הפעילה).

## דברים שכדאי לדעת לפני הבדיקה

- בסיס הנתונים הוא קבצי JSON תחת `server/data/` (אין צורך בהתקנת DB).
- הרשמה כבייביסיטר מוסיפה את המשתמש גם ל-`sitters.json`, כך שהוא מופיע מיד בדף החיפוש.
- אימות משתמשים מבוסס JWT (`Authorization: Bearer <token>`). ה-token מתקבל מ-Register או Login.
- קובץ `Babysitter.postman_collection.json` כולל את כל ה-routes עם דוגמאות בקשה/תשובה תקינה ושגיאה. יש להגדיר משתנה סביבה `baseUrl=http://localhost:3000`.

## משתמשי בדיקה

ניתן ליצור משתמש חדש דרך דף ההרשמה, או דרך בקשת ה-Register באוסף ה-Postman.
